# Миграция строк счёта для старых однострочных счетов

Доработка карточки заказа (формат/финансы/автосохранение/строки счёта)
заменила у счёта (`Document.type=INVOICE`) единственные `amount` +
`serviceDescription` на список `InvoiceLineItem`. Этот скрипт переносит уже
существующие старые счета под новую модель — без обращения к какому-либо
внешнему источнику, всё нужное уже в базе.

## Что делает

Для каждого счёта, у которого ещё нет ни одной строки и заполнена сумма,
создаёт РОВНО одну строку:

- `description` — существующий `serviceDescription`, если заполнен, иначе
  запасной текст «Услуга (перенесено из старой формы счёта)»;
- `quantity = 1`, `unit = SERVICE`;
- `unitPrice` — текущее значение `Document.amount`;
- `vatRate = NOT_APPLICABLE`;
- `migratedFromLegacyAmount = true` — отличает перенесённую строку от
  введённых вручную построчно.

## Что НЕ делается

- `Document.amount` не меняется — он и так уже верный, строка лишь объясняет
  задним числом, откуда взялась уже существующая сумма.
- Не пытается угадать реальный состав старого счёта (несколько позиций,
  скидки и т.п.) — восстановить это из одного числа невозможно, отсюда одна
  обобщённая строка, а не несколько.
- Счета, у которых уже есть хотя бы одна строка (из прошлого запуска или
  введена вручную через «Документы» → «Счета»), не трогаются.
- Счета без суммы (`amount IS NULL`) — переносить нечего, пропускаются.

## Команды

```bash
set -a && source .env.local && set +a

npx tsx scripts/migrate-invoice-line-items/dry-run.ts     # ничего не меняет, только отчёт
npx tsx scripts/migrate-invoice-line-items/apply.ts        # применяет план, пишет манифест для отката
npx tsx scripts/migrate-invoice-line-items/rollback.ts scripts/migrate-invoice-line-items/backups/apply-....json
```

Перед реальным запуском `apply.ts` на проде — `npm run db:backup` (единственная
база в проекте одновременно dev и prod, см. AGENTS.md, "Data Safety and Audit
Integrity").

## Идемпотентность

`buildPlan()` каждый раз считает план заново от текущего состояния базы
(есть ли у счёта строки) — уже перенесённые счета просто не попадают в план
как `create` при повторном запуске.

## Откат

`apply.ts` пишет манифест в `scripts/migrate-invoice-line-items/backups/apply-<дата>.json`
со списком созданных строк. `rollback.ts` с этим файлом удаляет их — по
одной, каждую сверяя `findFirst` с условием `migratedFromLegacyAmount: true`
перед удалением, чтобы не задеть строку, если её успели отредактировать
вручную после миграции (см. AGENTS.md, "Data Safety and Audit Integrity", п.2).
`Document.amount` откат не трогает — `apply.ts` его и не менял.
